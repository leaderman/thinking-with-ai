# 飞书图床：本地图片上传并替换为临时访问链接手册

## 目标

将 Markdown 文件中的本地图片路径（如 `article-images/img-00.jpg`），替换为飞书云文档中图片的临时访问链接，使文档可在任意环境渲染图片。

---

## 工具依赖

- `lark-cli`（飞书命令行工具，需已完成应用配置）
- `Node.js`（用于批量处理和替换）
- 飞书账号已通过 `lark-cli auth login` 授权 `drive` 相关 scope

---

## 完整流程

### 第一步：确认 user 身份授权有效

```bash
lark-cli auth status
```

检查 `tokenStatus` 是否为 `valid`，以及是否包含以下 scope：
- `drive:file:upload`
- `docs:document.media:upload`
- `docs:document.content:read`

如果 token 已过期（`tokenStatus: expired`），重新授权：

```bash
lark-cli auth login --domain drive --no-wait
# 将输出的 verification_url 发给用户在浏览器中打开授权
# 授权完成后执行：
lark-cli auth login --device-code <device_code>
```

> ⚠️ `auth login` 不支持 `--as` 参数，去掉即可。

---

### 第二步：找到「图床」文件夹的 folder_token

```bash
lark-cli drive files list
```

在返回结果中找到目标文件夹（如「图床」），记录其 `token` 字段，即 `folder_token`。

---

### 第三步：在图床文件夹中创建云文档

```bash
lark-cli docs +create --title "文章标题-images" --folder-token <folder_token> --markdown "图床文档"
```

记录返回值中的 `doc_id`（格式如 `ZJ0JdkHHho9HG0x8eMwcT4wxnAe`）。

> ⚠️ 后续上传图片时使用 `doc_id`，不要使用 `doc_url`（尤其是 wiki 形式的 URL 会导致报错）。

---

### 第四步：上传本地图片到云文档

使用 `docs +media-insert` 逐张上传。

**关键注意事项：**

1. **必须在图片所在目录下执行**，不能使用绝对路径：
   ```bash
   cd /path/to/images
   lark-cli docs +media-insert --doc <doc_id> --file ./img-00.jpg
   ```
   使用绝对路径会报错：`unsafe file path: --file must be a relative path within the current directory`

2. **进度信息输出到 stderr，JSON 结果输出到 stdout**，批量时用 `2>/dev/null` 过滤进度：
   ```bash
   result=$(lark-cli docs +media-insert --doc <doc_id> --file ./img-00.jpg 2>/dev/null)
   token=$(echo "$result" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['data']['file_token'])")
   ```

3. **每次上传返回一个 `file_token`**，需记录「文件名 → file_token」的映射关系，保存到文本文件备用：
   ```
   img-00.jpg Y1dQbv9oVoehIJxiqWIc8gTcnEc
   img-01.jpg QcVJbf3QKo5VMxxEjflcek8fnIf
   ...
   ```

批量上传示例（Shell 脚本）：

```bash
#!/bin/bash
DOC=<doc_id>
TOKENS_FILE="./tokens.txt"
> "$TOKENS_FILE"  # 清空

for i in $(seq -w 0 25); do
  ext="jpg"
  file="./img-${i}.${ext}"
  [ ! -f "$file" ] && ext="png" && file="./img-${i}.${ext}"
  
  token=$(lark-cli docs +media-insert --doc "$DOC" --file "$file" 2>/dev/null \
    | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['data']['file_token'])")
  
  echo "img-${i}.${ext} $token" >> "$TOKENS_FILE"
  echo "img-${i}: $token"
done
```

### 第五步：获取临时访问链接

使用飞书 API `batch_get_tmp_download_url` 将 `file_token` 转换为可访问的临时 URL。

**关键注意事项：**

1. **每批最多 5 个 token**，超过 5 个会返回 `1061002 params error`：
   ```bash
   # ✅ 正确：最多 5 个
   lark-cli api GET /open-apis/drive/v1/medias/batch_get_tmp_download_url \
     --params '{"file_tokens":["token1","token2","token3","token4","token5"]}'
   
   # ❌ 错误：超过 5 个会报 params error
   ```

2. **`--params` 必须是单行 JSON**，多行 JSON 会导致解析失败

3. **lark-cli api 命令的输出在 stderr**，重定向到文件时需用 `> file 2>&1`（注意顺序）

4. **临时链接有效期约 24 小时**，过期后需重新调用此接口获取新链接

批量获取示例（Node.js）：

```javascript
const { execSync } = require('child_process');

function getUrls(tokens) {
  const urlMap = {};
  const BATCH = 5;
  for (let i = 0; i < tokens.length; i += BATCH) {
    const batch = tokens.slice(i, i + BATCH);
    const params = JSON.stringify({ file_tokens: batch });
    const raw = execSync(
      `lark-cli api GET /open-apis/drive/v1/medias/batch_get_tmp_download_url --params '${params}'`,
      { encoding: 'utf8' }
    );
    const result = JSON.parse(raw);
    result.data.tmp_download_urls.forEach(({ file_token, tmp_download_url }) => {
      urlMap[file_token] = tmp_download_url;
    });
  }
  return urlMap;
}
```

### 第六步：替换 Markdown 中的本地路径

读取「文件名 → file_token」映射 + 「file_token → URL」映射，组合出「本地路径 → URL」替换关系，写入 Markdown 文件：

```javascript
const fs = require('fs');

// 读取 tokens.txt，构建 filename -> token 映射
const lines = fs.readFileSync('./tokens.txt', 'utf8').trim().split('\n');
const fileTokenMap = {};
lines.forEach(line => {
  const [filename, token] = line.trim().split(' ');
  fileTokenMap[filename] = token;
});

// 获取临时 URL（见第五步）
const urlMap = getUrls(Object.values(fileTokenMap));

// 替换 Markdown
let md = fs.readFileSync('article.md', 'utf8');
Object.entries(fileTokenMap).forEach(([filename, token]) => {
  const url = urlMap[token];
  if (url) md = md.replaceAll(`article-images/${filename}`, url);
});
fs.writeFileSync('article.md', md, 'utf8');
```

## 常见报错速查

| 报错信息 | 原因 | 解决方案 |
|----------|------|----------|
| `unsafe file path: --file must be a relative path` | 使用了绝对路径 | `cd` 到图片目录，改用 `./img-00.jpg` |
| `API error: [1061002] params error` | 单次请求 token 数量超过 5 个，或 JSON 多行 | 每批最多 5 个，使用单行 JSON |
| `API call failed: EOF` | user token 过期 | 重新执行 `lark-cli auth login --domain drive` |
| `unknown flag: --as` | `auth login` 不支持 `--as` | 去掉 `--as user` 参数 |
| `json.decoder.JSONDecodeError` | 解析时混入了进度输出（stderr） | 加 `2>/dev/null` 过滤 stderr |
| 临时链接打开 403 | 链接已过期（有效期约 24 小时） | 重新调用 `batch_get_tmp_download_url` 获取新链接 |

---

## 关于临时链接的说明

飞书 `batch_get_tmp_download_url` 返回的是**有时效的临时链接**，格式为：
```
https://internal-api-drive-stream.feishu.cn/space/api/box/stream/download/authcode/?code=...
```

- 有效期约 **24 小时**
- 适合一次性导出、预览场景
- 如需**永久稳定链接**，考虑将图片上传到对象存储（OSS/COS）或其他支持永久外链的图床
